# Livewire Integration Guide

This document explains how Livewire was integrated into the Caféra project to enable dynamic, real-time interactivity.

## 1. Installation and Configuration

Livewire was installed via Composer:
```bash
composer require livewire/livewire
```

### Layout Modification
To support Livewire across the dashboard, the main layout file [resources/views/dashboard/layout.blade.php](file:///c:/xampp/htdocs/CoffeeShop/resources/views/dashboard/layout.blade.php) was updated:

- **Styles**: `@livewireStyles` was added inside the `<head>` tag.
- **Scripts**: `@livewireScripts` was added before the closing `</body>` tag.

## 2. Component Architecture

### Component Class
Components are located in `app/Livewire/Dashboard/`. For example, the Menu component:
- **File**: `app/Livewire/Dashboard/Menu.php`
- **Purpose**: Handles state logic for search, category filtering, and cart management.

### Component View
Blade views for components are located in `resources/views/livewire/dashboard/`.
- **File**: `resources/views/livewire/dashboard/menu.blade.php`
- **Interactivity**: Uses `wire:model` and `wire:click` for real-time updates without page reloads.

## 3. Modular View Structure

To maintain a clean and DRY (Don't Repeat Yourself) architecture, the dashboard views were refactored:

1.  **Base Layout**: All dashboard views now `@extends('dashboard.layout')`.
2.  **Boilerplate Removal**: Redundant `<html>`, `<head>`, and script tags were removed from individual views (e.g., `dashboard.blade.php`, `orders.blade.php`) to avoid asset duplication.
3.  **Livewire Routing**: Routes in `routes/web.php` were updated to use Livewire component classes directly:
    ```php
    Route::get('dashboard/menu', \App\Livewire\Dashboard\Menu::class);
    ```

## 4. Best Practices for New Components

When adding new interactive features:
1.  **Generate a Component**: `php artisan make:livewire Dashboard/NewFeature`
2.  **Use the Layout**: Ensure the component uses `->layout('dashboard.layout')` in its `render()` method.
3.  **Bind Inputs**: Use `wire:model.live` for real-time searching or `wire:model.blur` for form fields to optimize performance.
